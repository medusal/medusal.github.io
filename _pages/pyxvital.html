Pyxistem procédure installation Windows 10 :

1. Décompresser l'archive 'Pyxvital_167_dev01.zip' dans un répertoire de votre choix puis exécutez 'setup.exe'

NE PAS LANCER PYVXVITAL avant la fin de cette procédure

2. Installer fsv-1.40.1105-Signe.msi

3. Ajouter les mots clés suivants pour désactiver le contrôle de la signature CPS dans le fichier "Sesam.ini" :

C:\ProgramData\santesocial\fsv\1.40.11\conf

[SSV]

DesactiverControleSignatureCPSTest=1

CertificatsDebug=1

4. Copier le répertoire "\certificats-debug" présent dans le package d'installation dans le chemin suivant:

C:\ Program Files (x86)\santesocial\fsv\1.40.11\ssv

5. Décompresser l'archive 'Galss_cryptolib.zip' dans un répertoire de votre choix puis exécutez dans l'ordre "Galss" puis "CryptolibCPS-5.0.24.msi" Démarrer "Pyxvital.exe"

Erreurs possibles si CPS non reconnue :

- GALSS.INI dans répertoire windows vide : le supprimer et réinstaller le GALSS

- GALSS.INI dans répertoire \Users\xxxxxx\AppData\Local\VirtualStore\Windows vide : le supprimer

- Redémarrer Windows

- GALSS.INI dans répertoire Windows user n’a pas tous les droits : lui mettre tous les droits

6. HTTPS : Ajouter les fichiers de l'archive Pyxcom_anonyme_https.zip au répertoire /Pyxvital existant.

Paramétrage de la connexion HTTPS via le fichier "Pyxcomserveur.ini":

Ajouter les lignes suivantes (port au choix) :

[Host]

Port=4443

[Anonyme]

Port=9080

Ajouter dans Pyxvital.ini partie [PARAMETRAGE] :

Web_serveur_port=9080

Serveur_Web=O

Le composant "PyxcomServeur.exe" doit être lancé localement soit par double-clique soit par ligne de commande.

Le mot de passe de la clé serveur Pyxcom est 'pyxistem'.

Avec la syntaxe ci-dessous le mot de passe peut être transmit au démarrage.

Pyxcomserveur.exe -noinput -password mon_mot_de_passe

Soit

Pyxcomserveur.exe -noinput -password pyxistem

Les requêtes Https sont transmissent en direction de PyxcomServeur (443 par défaut).

Chrome :
